
## 1. 敏感数据扫描阶段

### TC-SEC-01-01: 扫描表中的敏感数据 🔴

**目的**: 验证可以扫描表中的敏感数据字段

**前置条件**: Sensitive Detect 服务已启动，数据库中存在测试表

**测试步骤**:
```bash
TOKEN=$(curl -s -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

curl -X POST http://localhost:8015/api/sensitive/scan \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "table_name": "customers",
    "sample_size": 100
  }'
```

**预期结果**:
- 状态码: 200
- 响应包含:
  - `id`: 报告 ID
  - `table_name`: 扫描的表名
  - `scan_time`: 扫描时间
  - `total_columns`: 总列数
  - `sensitive_columns`: 敏感列数
  - `fields`: 敏感字段详情列表
  - `risk_level`: 整体风险级别

**响应示例**:
```json
{
  "id": "abc12345",
  "table_name": "customers",
  "scan_time": "2024-01-29T10:00:00Z",
  "total_columns": 10,
  "sensitive_columns": 3,
  "fields": [
    {
      "column_name": "phone",
      "sensitivity_level": "high",
      "detected_types": ["phone"],
      "detection_method": "field_name+regex",
      "sample_count": 95,
      "confidence": 0.95
    },
    {
      "column_name": "id_card",
      "sensitivity_level": "critical",
      "detected_types": ["id_card"],
      "detection_method": "regex",
      "sample_count": 88,
      "confidence": 0.88
    }
  ],
  "risk_level": "critical"
}
```

### TC-SEC-02-01: 使用 LLM 分类敏感数据 🔴

**目的**: 验证可以使用 LLM 对复杂数据进行敏感性分类

```bash
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

curl -X POST http://localhost:8015/api/sensitive/classify \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "data_samples": [
      "张三，男，1990年1月1日出生，住址：北京市朝阳区xxx路xxx号",
      "李四的联系电话是13800138000，紧急联系人王五 13900139000"

    "context": "客户信息表"
  }'
```

**预期结果**:
- 状态码: 200
- 响应包含 AI 分析结果
- 识别出姓名、日期、地址、电话等敏感信息

### TC-SEC-08-02: 敏感数据风险评估 🟡

```bash
  -H "Content-Type: application/json" \
```
