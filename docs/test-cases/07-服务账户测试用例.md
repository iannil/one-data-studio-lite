> **文档版本**: v2.0
> **更新日期**: 2026-01-31

## 用户生命周期流程

```
创建 → 密钥生成 → 权限配置 → 服务调用 → 密钥轮换 → 监控审计 → 禁用/删除
```

## 1. 账号创建阶段

### TC-SVC-01-01-01: 创建服务账户 🔴

**目的**: 验证服务账户创建流程正常

**前置条件**:
- Portal 服务已启动
- 已存在 service_account 角色

**测试步骤**:
```bash
# 由超级管理员创建
curl -X POST http://localhost:8010/api/service-accounts \
  -H "Authorization: Bearer $SUPER_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "data_sync_service",
    "description": "数据同步服务账户",
    "permissions": ["data:read", "service:call"],
    "ip_whitelist": ["10.0.0.10", "10.0.0.11"]
  }'
```

**预期结果**:
- 状态码: 201
- 响应包含:
  - `client_id`: 服务账户 ID
  - `client_secret`: 生成的密钥（仅显示一次）

## 3. 功能使用-服务调用阶段

### TC-SVC-04-06-01: Data API 调用-使用 Secret 🔴

**目的**: 验证服务账户可以使用 Secret 调用 Data API

**前置条件**:
- 已创建服务账户
- 已获取 client_id 和 client_secret

**测试步骤**:
```bash
# 方法1: 使用 Basic Auth
curl -X GET "http://localhost:8014/api/data/sales_data?page=1&page_size=10" \
  -u "data_sync_service:CLIENT_SECRET"

# 方法2: 使用服务账户 Token
curl -X POST http://localhost:8010/api/service-accounts/token \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "data_sync_service",
    "client_secret": "CLIENT_SECRET"
  }' | jq -r '.token'
```

**预期结果**:
- 状态码: 200
- 成功获取数据
