## 7. 综合使用场景

### TC-USR-07-01: 业务用户日常工作流程 🟡

**目的**: 验证业务用户完整的日常使用流程

**前置条件**: 所有相关服务已启动

```bash
# 1. 登录

# 2. 搜索数据资产
echo "=== 搜索数据资产 ==="
```

### TC-USR-07-02: 数据探索到查询完整流程 🟡

**目的**: 验证从数据探索到查询的完整流程

**前置条件**: 所有相关服务已启动

**测试步骤**:
```bash
TOKEN=$(curl -s -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

# 1. 查看可用表
echo "=== 可用数据表 ==="
TABLES=$(curl -s -X GET http://localhost:8011/api/nl2sql/tables \
  -H "Authorization: Bearer $TOKEN")
echo $TABLES | jq '.[0:3] | .[] | .table_name'

# 2. 查看第一个表的结构
FIRST_TABLE=$(echo $TABLES | jq -r '.[0].table_name')
echo "=== 表 $FIRST_TABLE 的结构 ==="
curl -s -X GET "http://localhost:8014/api/data/$FIRST_TABLE/schema" \
  -H "Authorization: Bearer $TOKEN" | jq '.columns[] | {name, data_type}'

# 3. 预览数据
echo "=== 数据预览 ==="
curl -s -X GET "http://localhost:8014/api/data/$FIRST_TABLE?page_size=3" \
  -H "Authorization: Bearer $TOKEN" | jq '.data'

# 4. 自然语言查询
echo "=== 自然语言查询 ==="
curl -s -X POST http://localhost:8011/api/nl2sql/query \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"question\": \"从 $FIRST_TABLE 表中查询前5条记录\"}" | jq '{generated_sql, rows}'
```

**预期结果**:
- 完成从探索到查询的完整流程
- 各步骤数据一致
