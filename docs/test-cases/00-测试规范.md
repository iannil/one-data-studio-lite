## 二、测试用例编写规范

### 2.1 用例编号规范

```
TC-{角色代码}-{生命周期阶段}-{功能模块}-{序号}
```

**规则说明**:
- **TC**: 前缀，固定为 "TC" (Test Case)
- **角色代码**: 3 位大写字母（见 2.2 节）
- **生命周期阶段**: 2 位数字（01-09）
- **功能模块**: 2 位数字（01-15）
- **序号**: 2 位数字（01-99）

**示例**:
- `TC-SUP-01-01-01`: 超级管理员-账号创建-认证-创建超级管理员账号
- `TC-ADM-04-08-01`: 管理员-功能使用-Audit Log-查询审计日志

### 2.2 角色代码定义

| 代码 | 角色名称 | 英文名称 |
|------|----------|----------|
| COM | 通用 | Common |
| SUP | 超级管理员 | Super Admin |
| ADM | 管理员 | Admin |
| SCI | 数据科学家 | Data Scientist |
| ANA | 数据分析师 | Data Analyst |
| VW | 查看者 | Viewer |
| SVC | 服务账户 | Service Account |
| XROLE | 跨角色 | Cross-Role |

### 2.3 生命周期阶段编号

| 编号 | 阶段名称 | 描述 |
|------|----------|------|
| 01 | 账号创建 | 创建用户账号、首次登录、初始化设置 |
| 02 | 权限配置 | 分配权限、角色配置、访问控制 |
| 03 | 数据访问 | 数据查询、Schema 获取、资产浏览 |
| 04 | 功能使用 | 各角色核心功能使用场景 |
| 05 | 监控审计 | 查看操作日志、审计报告、统计信息 |
| 06 | 维护操作 | 密钥轮换、资源管理、配置更新 |
| 07 | 账号禁用 | 禁用账号、权限回收、访问阻断 |
| 08 | 账号删除 | 删除账号、数据清理、归档 |
| 09 | 紧急操作 | 紧急响应、故障处理、系统恢复 |

## 三、测试用例内容规范

每个测试用例必须包含以下要素：

### 3.1 必需字段

| 字段 | 说明 | 示例 |
|------|------|------|
| **用例编号** | 遵循 2.1 节规范 | TC-SUP-01-01-01 |
| **用例标题** | 简洁描述测试内容 | 创建超级管理员账号 |
| **优先级** | P0/P1/P2/P3 | P0 |
| **目的** | 测试目标说明 | 验证超级管理员账号创建流程正常工作 |
| **前置条件** | 执行测试前的环境/状态要求 | Portal 服务已启动 |
| **测试步骤** | 详细操作步骤 | 1. 调用接口 xxx<br>2. 验证响应 yyy |
| **预期结果** | 明确的预期输出 | 状态码 200，返回 token |
| **响应示例** | (可选) 预期的响应 JSON | {...} |

### 3.2 用例模板

```markdown
### TC-{角色代码}-{阶段}-{模块}-{序号}: {用例标题} {优先级图标}

**目的**: {测试目的说明}

**前置条件**: {前置条件列表}

**测试步骤**:
```bash
# 示例命令或操作步骤
curl -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}'
```

**预期结果**:
- 状态码: 200
- 响应包含:
  - `success`: true
  - `token`: 非空 JWT 字符串
  - `user.user_id`: "admin"
  - `user.role`: "super_admin"

**响应示例**:
```json
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "user_id": "admin",
    "username": "admin",
    "role": "super_admin",
    "display_name": "超级管理员"
  },
  "message": "登录成功"
}
```
```

## 四、测试优先级规范

### 4.1 优先级定义

| 级别 | 说明 | 图标 | 覆盖要求 |
|------|------|------|----------|
| **P0** | 核心功能，必须通过 | 🔴 | 100% 覆盖，每次发布必测 |
| **P1** | 重要功能 | 🟡 | 100% 覆盖，每次发布必测 |
| **P2** | 一般功能 | 🟢 | 主干版本覆盖 |
| **P3** | 边界/异常测试 | ⚪ | 主干版本覆盖 |

### 4.2 优先级判定标准

**P0 用例**:
- 用户登录/登出
- 核心业务流程（如数据查询）
- 安全认证
- 关键数据修改

**P1 用例**:
- 主要功能入口
- 数据导出
- 系统配置修改
- 权限验证

**P2 用例**:
- 次要功能
- 辅助性功能
- UI 交互细节

**P3 用例**:
- 边界条件测试
- 异常输入测试
- 性能测试
- 兼容性测试

## 六、pytest 标记规范

```python
@pytest.mark.super_admin      # 超级管理员测试

@pytest.mark.data_scientist   # 数据科学家测试

@pytest.mark.viewer           # 查看者测试

### 6.2 优先级标记

```python
@pytest.mark.p0   # P0 优先级
@pytest.mark.p1   # P1 优先级
@pytest.mark.p2   # P2 优先级
@pytest.mark.p3   # P3 优先级

```python

## 七、测试执行规范

### 7.1 测试执行流程

1. **环境准备**
   - 确保所有服务启动
   - 检查数据库连接
   - 准备测试数据

2. **执行测试**
   - 按优先级执行：P0 → P1 → P2 → P3
   - 记录测试结果
   - 收集失败信息

3. **结果报告**
   - 生成测试报告
   - 分析失败原因
   - 提交缺陷报告

### 7.2 命令示例

```bash
# 运行所有 P0 测试
pytest -m p0 -v

# 运行特定角色的测试
pytest -m super_admin -v

# 运行集成测试
pytest -m integration -v

# 排除慢速测试
pytest -m "not slow" -v

# 生成覆盖率报告
pytest --cov=services --cov-report=html
```

## 八、测试用例维护规范

### 8.1 用例更新时机

- 功能新增/变更时
- 发现用例缺陷时
- 测试环境变更时
- 定期审查优化

### 8.2 用例作废规范

- 功能下线时标注用例作废
- 保留作废用例历史记录

### 8.3 版本控制

- 每次用例修改更新变更记录
- 记录修改日期和修改内容
- 重要修改需要评审

## 十、附录

### 10.1 HTTP 状态码参考

| 状态码 | 说明 | 使用场景 |
|--------|------|----------|
| 200 | 成功 | 请求成功 |
| 201 | 已创建 | 资源创建成功 |
| 400 | 请求错误 | 参数错误、非法操作 |
| 401 | 未授权 | Token 无效/过期 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 未找到 | 资源不存在 |
| 422 | 无法处理 | 验证失败 |
| 500 | 服务器错误 | 内部错误 |
| 503 | 服务不可用 | 外部服务故障 |

### 10.2 常用 cURL 命令模板

```bash
# GET 请求
curl -X GET http://localhost:8010/api/endpoint \
  -H "Authorization: Bearer $TOKEN"

# POST 请求
curl -X POST http://localhost:8010/api/endpoint \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"key": "value"}'

# PUT 请求
curl -X PUT http://localhost:8010/api/endpoint \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"key": "value"}'

# DELETE 请求
curl -X DELETE http://localhost:8010/api/endpoint \
  -H "Authorization: Bearer $TOKEN"
```
