### TC-ENG-02-02: 元数据变更触发 ETL 任务 🟡

**目的**: 验证元数据变更可以自动触发关联的 ETL 任务

**前置条件**: Metadata Sync 服务已启动，已配置映射规则

**测试步骤**:
```bash
TOKEN=$(curl -s -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

# 1. 先创建映射规则
curl -X PUT http://localhost:8013/api/metadata/mappings/mapping-001 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "mapping-001",
    "source_urn": "urn:li:dataset:(urn:li:dataPlatform:mysql,test_db.users,PROD)",
    "target_task_type": "dolphinscheduler",
    "target_task_id": "100",
    "trigger_on": ["CREATE", "UPDATE"],
    "enabled": true
  }'

# 2. 发送变更事件
curl -X POST http://localhost:8013/api/metadata/webhook \
  -H "Content-Type: application/json" \
  -d '{
    "entity_urn": "urn:li:dataset:(urn:li:dataPlatform:mysql,test_db.users,PROD)",
    "change_type": "UPDATE",
    "changed_fields": ["schema"],
    "timestamp": "2024-01-29T10:00:00Z"
  }'
```

**预期结果**:
- 状态码: 200
- 响应包含:
  - `affected_tasks`: ["dolphinscheduler:100"]

### TC-ENG-03-01: 分析表的数据质量 🔴

**目的**: 验证可以分析指定表的数据质量问题

```bash
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

curl -X POST http://localhost:8012/api/cleaning/analyze \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "table_name": "test_users",
    "sample_size": 1000
  }'
```

**预期结果**:
- 状态码: 200
- 响应包含:
  - `table_name`: "test_users"
  - `total_rows`: 表总行数
  - `analyzed_rows`: 分析的行数
  - `issues`: 质量问题列表
  - `quality_score`: 0-100 的质量得分

**响应示例**:
```json
{
  "table_name": "test_users",
  "total_rows": 10000,
  "analyzed_rows": 1000,
  "issues": [
    {
      "column": "email",
      "issue_type": "null_values",
      "description": "字段 email 存在 150 个空值",
      "affected_rows": 150,
      "severity": "high"
    },
    {
      "column": "phone",
      "issue_type": "duplicates",
      "description": "字段 phone 存在重复值",
      "affected_rows": 50,
      "severity": "low",
      "sample_values": ["13800138000", "13900139000"]
    }
  ],
  "quality_score": 75
}
```

### TC-ENG-04-01: 获取 AI 推荐的清洗规则 🔴

```bash
    "sample_size": 500
  }'
```

**预期结果**:

  - `description`: 规则描述

  - `rule_type`: 规则类型
  - `config`: 配置参数

**响应示例**:
```json
{
  "rules": [
    {
      "rule_id": "abc123",
      "name": "过滤email空值",
      "description": "过滤email字段为空的记录",
      "target_column": "email",
      "rule_type": "filter",
      "config": {"condition": "is_not_null"}

    {
      "rule_id": "def456",
      "name": "phone去重",

      "target_column": "phone",
      "rule_type": "deduplicate",
      "config": {}
    }
  ],
  "explanation": "基于数据质量分析，发现 2 个问题，推荐 2 条清洗规则"
}
```
