## 2. 认证授权测试

### TC-COM-02-01: 正常登录 🔴

**目的**: 验证用户使用正确的用户名和密码可以成功登录

**前置条件**: Portal 服务已启动

**测试步骤**:
```bash
curl -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}'
```

**预期结果**:
- 状态码: 200
- 响应包含:
  - `success`: true
  - `token`: 非空 JWT 字符串
  - `user.user_id`: "admin"
  - `user.role`: "admin"
  - `message`: "登录成功"

**响应示例**:
```json
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "user_id": "admin",
    "username": "admin",
    "role": "admin",
    "display_name": "管理员"
  },
  "message": "登录成功"
}
```

## 5. 审计日志中间件测试

### TC-COM-05-01: API 调用自动记录审计日志 🟡

**目的**: 验证 API 调用会自动记录到审计日志

**前置条件**: 所有服务已启动

**测试步骤**:
```bash
# 1. 登录
TOKEN=$(curl -s -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

# 2. 执行一个 API 调用
curl -X GET http://localhost:8011/api/nl2sql/tables \
  -H "Authorization: Bearer $TOKEN"

# 3. 等待审计日志记录
sleep 1

# 4. 查询审计日志
curl -X GET "http://localhost:8016/api/audit/logs?subsystem=nl2sql" \
  -H "Authorization: Bearer $TOKEN"
```

**预期结果**:
- 审计日志中包含刚才的 API 调用记录
- 记录包含:
  - `subsystem`: "nl2sql"
  - `event_type`: "api_call"
  - `user`: "admin"
  - `action`: "GET /api/nl2sql/tables"
  - `status_code`: 200

---

### TC-COM-05-02: 健康检查不记录审计日志 🟢

**目的**: 验证健康检查接口不会产生审计日志

**前置条件**: 所有服务已启动

**测试步骤**:
```bash
# 1. 登录获取 Token
TOKEN=$(curl -s -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

# 2. 记录当前审计日志数量
BEFORE=$(curl -s -X GET "http://localhost:8016/api/audit/stats" \
  -H "Authorization: Bearer $TOKEN" | jq '.total_events')

# 3. 多次调用健康检查
for i in {1..5}; do
  curl -s http://localhost:8011/health > /dev/null
done

# 4. 检查审计日志数量
AFTER=$(curl -s -X GET "http://localhost:8016/api/audit/stats" \
  -H "Authorization: Bearer $TOKEN" | jq '.total_events')

echo "Before: $BEFORE, After: $AFTER"
```

**预期结果**:
- 健康检查不会增加审计日志数量（或仅增加 stats 接口的调用记录）
