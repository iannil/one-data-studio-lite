# 超级管理员测试用例

> **文档版本**: v2.0
> **更新日期**: 2026-01-31
> **适用角色**: SUP (超级管理员)
> **用例数量**: 45

本文档包含超级管理员 (Super Admin) 的全生命周期测试用例。

---

## 用户生命周期流程

```
账号创建 → 权限分配 → 系统初始化 → 用户管理 → 角色管理 → 系统配置
    ↓
审计监督 → 紧急响应 → 账号转移/删除
```

## 1. 账号创建阶段

### TC-SUP-01-01-01: 创建超级管理员账号 🔴

**目的**: 验证超级管理员账号创建流程正常

**前置条件**:
- Portal 服务已启动
- 数据库已初始化

**测试步骤**:
```bash
curl -X POST http://localhost:8010/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "super_admin",
    "password": "admin123",
    "role": "super_admin",
    "display_name": "超级管理员"
  }'
```

**预期结果**:
- 状态码: 201
- 响应包含创建的用户信息和 token

### TC-SUP-01-01-03: 超级管理员修改默认密码 🟡

**目的**: 验证超级管理员可以修改密码

**前置条件**:
- 已登录超级管理员账号

**测试步骤**:
```bash
TOKEN=$(curl -s -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "super_admin", "password": "admin123"}' | jq -r '.token')

curl -X POST http://localhost:8010/auth/change-password \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "old_password": "admin123",
    "new_password": "NewSecurePass123!"
  }'
```

**预期结果**:
- 状态码: 200
- 响应: `{"success": true, "message": "密码修改成功"}`
