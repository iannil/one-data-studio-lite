### TC-STW-04-02: 监控多表数据质量 🟡

**目的**: 批量检查多个数据表的质量

**前置条件**: AI Cleaning 服务已启动，存在多个数据表

**测试步骤**:
```bash
TOKEN=$(curl -s -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

TABLES=("customers" "orders" "products")

for table in "${TABLES[@]}"; do
  echo "Analyzing $table..."
  curl -s -X POST http://localhost:8012/api/cleaning/analyze \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d "{\"table_name\": \"$table\", \"sample_size\": 500}" | jq '{table: .table_name, score: .quality_score, issues: .issues | length}'
done
```

**预期结果**:
- 每个表都返回质量分析结果
- 可以比较各表的质量得分

### TC-STW-07-02: 数据质量问题处理流程 🟡

**目的**: 验证发现数据质量问题后的处理流程

**前置条件**: AI Cleaning 服务已启动

**测试步骤**:
```bash
TOKEN=$(curl -s -X POST http://localhost:8010/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}' | jq -r '.token')

# 1. 分析发现问题
REPORT=$(curl -s -X POST http://localhost:8012/api/cleaning/analyze \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"table_name": "problem_table", "sample_size": 500}')

echo "Quality Score: $(echo $REPORT | jq '.quality_score')"
echo "Issues: $(echo $REPORT | jq '.issues')"

# 2. 获取清洗建议
RECOMMEND=$(curl -s -X POST http://localhost:8012/api/cleaning/recommend \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"table_name": "problem_table", "sample_size": 500}')

echo "Recommended Rules: $(echo $RECOMMEND | jq '.rules')"

# 3. 生成 SeaTunnel 配置
RULES=$(echo $RECOMMEND | jq '.rules')
curl -X POST http://localhost:8012/api/cleaning/generate-config \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"table_name\": \"problem_table\", \"rules\": $RULES}"
```

**预期结果**:
- 完整的问题发现到解决方案生成流程
- 生成可用的清洗配置
