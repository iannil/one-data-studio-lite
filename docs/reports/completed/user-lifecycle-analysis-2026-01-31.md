# 用户全生命周期分析文档

> **文档版本**: v1.0
> **创建日期**: 2026-01-31
> **作者**: Claude Code
> **用途**: 梳理不同用户的全生命周期，确保所有功能被覆盖

---

## 一、文档概述

本文档基于 ONE-DATA-STUDIO-LITE 智能大数据平台的需求描述和测试用例，梳理了所有用户角色的全生命周期流程。通过分析用户从创建到删除的完整旅程，确保平台的所有功能在用户生命周期中得到覆盖。

---

## 二、用户角色体系

平台定义了 **6 种用户角色**，每种角色具有不同的权限范围和功能访问能力：

| 角色代码 | 角色名称 | 英文名称 | 权限级别 | 主要职责 |
|----------|----------|----------|----------|----------|
| SUP | 超级管理员 | Super Admin | 最高 | 系统全面管理、角色创建、紧急响应 |
| ADM | 管理员 | Admin | 高 | 用户管理、平台配置、审计查看 |
| SCI | 数据科学家 | Data Scientist | 中高 | 模型开发、数据分析、ETL 配置 |
| ANA | 数据分析师 | Data Analyst | 中 | 数据查询、报表制作、可视化 |
| VW | 查看者 | Viewer | 低 | 只读访问、报表查看 |
| SVC | 服务账户 | Service Account | 特殊 | 服务间通信、自动化调用 |

---

## 三、生命周期阶段定义

所有用户角色经历以下 **9 个生命周期阶段**：

| 编号 | 阶段名称 | 说明 | 关键操作 |
|------|----------|------|----------|
| 01 | 账号创建 | 创建用户账号、首次登录、初始化设置 | 注册、登录、修改密码、权限查看 |
| 02 | 权限配置 | 分配权限、角色配置、访问控制 | 权限申请、角色分配、资源申请 |
| 03 | 数据访问 | 数据查询、Schema 获取、资产浏览 | 数据查询、Schema 获取、资产搜索 |
| 04 | 功能使用 | 各角色核心功能使用场景 | NL2SQL、AI 清洗、Pipeline 操作 |
| 05 | 监控审计 | 查看操作日志、审计报告、统计信息 | 日志查询、统计查看、报告导出 |
| 06 | 维护操作 | 密钥轮换、资源管理、配置更新 | 密码修改、资源管理、配置更新 |
| 07 | 账号禁用 | 禁用账号、权限回收、访问阻断 | 账号禁用、权限撤销 |
| 08 | 账号删除 | 删除账号、数据清理、归档 | 账号删除、数据清理 |
| 09 | 紧急操作 | 紧急响应、故障处理、系统恢复 | 紧急停止、Token 撤销 |
